## 📖 核心知识点

今天围绕 **里程碑：博客 API v1** 建立可编码、可复述、可面试的理解。学习时不要只记 API 名称，要把“为什么这样设计、什么时候使用、常见错误是什么”写进笔记。

建议学习顺序：

1. 阅读官方文档或权威教程，先建立概念框架。
2. 跟写一个最小 Demo，确认环境和基础 API 可用。
3. 独立扩展 1-2 个小功能，避免只停留在复制代码。
4. 用自己的话整理概念、代码结构和易错点。

## 💻 实践练习

1. 完成博客文章、分类、评论的核心 REST API。
2. 接入参数校验、统一异常、统一响应和 OpenAPI 文档。
3. 至少为 3 个核心接口补充单元或集成测试。
4. 整理 README 和接口截图，形成可演示版本。
