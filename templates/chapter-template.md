# 章节四件套模板

> 新建章节时，把本文件四段分别复制到 `01-theory.md` / `02-demo.md` / `03-check.md` / `04-project-task.md`。
> 不要再粘「理解 X 时要同时关注三个问题」之类的模板套话——每章只保留独有内容。

---

## 01-theory.md 模板

```markdown
# Chapter {NN} {标题} - 理论

> 前置：[[{NN-1}-XXX]] [[YY-XXX]]
> 后续：[[{NN+1}-XXX]]
> 优先级：L{1|2|3}  预计：{N}小时

## 1. 为什么需要这一章
（一句话动机：解决什么真实问题）

## 2. 核心概念
- 概念 A：定义 + 一个例子
- 概念 B：定义 + 一个例子
（每个概念配链接：[[glossary#XXX]]）

## 3. 关键原理 / 工作机制
（图、流程、字节码、源码片段，必要时画 Mermaid）

## 4. 常见坑与最佳实践
- 坑 1：现象 → 根因 → 修复
- 坑 2：……

## 5. 与项目的衔接
本章在博客项目里承担什么角色：……
```

## 02-demo.md 模板

```markdown
# Chapter {NN} {标题} - 实操 Demo

## Demo 目标
（一句话：跑完后应该具备什么能力）

## 前置条件
- JDK 21 / Maven 3.9+ / 项目基线 pom（见 [[chapters/21-Maven工程化/02-demo]]）
- 新增依赖：……（只列增量，不重复基线）
- 数据：……

## 增量依赖
```xml
<dependency>
  <groupId>...</groupId>
  <artifactId>...</artifactId>
</dependency>
```

## 实操步骤
1. ……
2. ……

## 完整示例代码
（贴可直接复制运行的代码，不要伪代码）

## 运行与验证
| 检查项 | 命令 / 期望输出 |
|---|---|

## 常见坑
- ……
```

## 03-check.md 模板

```markdown
# Chapter {NN} {标题} - 验收

## 概念自测（5 题）
1. ……
2. ……

## 代码题（2 题）
- 题目 1：……
- 题目 2：……

## 费曼输出
用 5 分钟向一个完全不懂的人讲清「{本章核心概念}」，写下讲稿大纲。

## 参考答案
（每题给出标准答案与扣分点）

## 面试高频题（同步到 [[interview-bank]]）
- ⭐⭐ 题目：……
  - 答题要点：……
  - 🔥 追问：……

## 通过标准
- [ ] 概念题至少答对 4/5
- [ ] 两道代码题都能跑通
- [ ] 面试题能 2 分钟内口头作答
```

## 04-project-task.md 模板

```markdown
# Chapter {NN} {标题} - 项目任务

## 任务目标
（这个章节交付到博客项目的哪一部分）

## 任务清单（量化）
- [ ] 接口 / 类 / 方法名：……
- [ ] 配置项：……
- [ ] 测试用例：≥ N 个，覆盖 {正常 / 边界 / 异常}
- [ ] 日志：INFO 级别输出 `[{Module}] {action}: {key=val}`

## 验收清单（可勾选）
- [ ] 端到端跑通命令：……
- [ ] curl / Postman 验证：……
- [ ] 失败场景输出符合预期

## 扩展挑战（可选）
- ……

## 与下一章的衔接
本章产出会被 [[{NN+1}-XXX]] 用作 ……
```
