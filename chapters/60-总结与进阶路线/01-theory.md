# Chapter 60 总结与进阶路线 - 理论

> 前置：[[59-系统设计与面试]]
> 优先级：L1 必须掌握  预计：3 小时

## 1. 为什么需要这一章

学了 59 章后，你已经能写、能跑、能讲。 但能不能让面试官在 30 秒内决定「叫你来面」，关键在于**简历表达**和**项目包装**。 这一章把博客项目转化成一个 HR 与技术官都看得懂的「作品」。

## 2. 简历的 5 大模块

```
基本信息（姓名、岗位、联系方式、GitHub）
教育经历（学校 / 专业 / 时间，最近 → 最早）
项目经历（2 - 3 个，按重要度排序，每个 ≤ 100 字 + 4 个 bullet）
专业技能（按熟练度分层：精通 / 熟练 / 了解）
其他（开源 / 博客 / 比赛 / 证书，可选）
```

工作经验空缺时，**项目经历** + **GitHub 持续提交** 是最强信号。

## 3. STAR 模板（项目段落）

> **S**ituation 业务背景 + **T**ask 你的职责 + **A**ction 技术动作 + **R** esult 量化结果

### 通用骨架

```
[项目名]（YYYY.MM - YYYY.MM）｜ 个人项目 ｜ GitHub: https://github.com/...
一句话简介：用 [技术栈] 实现的 [业务定位] 系统，[规模 / 性能指标]。
- (A+R) 设计并实现 XXX，使 YYY 指标从 X 提升到 Y
- (A+R) 引入 XXX 解决 YYY 问题，避免了 ZZZ 风险
- (A) 基于 XXX 完成 YYY，包括 …
- (R) 项目上线后 XXX
```

### 博客 API v2 示范

```
个人博客 API（2025.06 - 2026.05）｜ 个人项目 ｜ github.com/xiong/blog-api
基于 Spring Boot 3.3 + MyBatis + MySQL + Redis 的博客后端，覆盖文章、评论、
认证授权、缓存、文件上传，本机压测详情接口 P99 < 50ms / QPS > 1500。

- 设计统一响应 + 全局异常 + 分层错误码体系，覆盖 5 类业务异常，定位时间下降 70%
- 实现 JWT Access(15m) + Refresh(7d) 双 Token + Redis 黑名单方案，
  支持主动登出与改密强制失效，相比单 Token 方案兼顾性能与安全
- 用 Redisson 分布式锁 + 空值哨兵 + 随机 TTL 解决缓存穿透/击穿/雪崩，
  压测 1000 并发查同一冷 key 时 DB 仅承担 1 次回源
- 多阶段 Dockerfile + docker-compose 一键启动 app+mysql+redis+rmq，镜像 220MB
- 单测覆盖率 75%，CI 接入 GitHub Actions，发版打 tag 自动构建镜像
```

## 4. 技能栏分层（务必诚实）

| 等级 | 含义 | 用词 |
|---|---|---|
| 精通 | 能讲原理 + 改过源码 / 工程优化 | 精通、熟悉原理 |
| 熟练 | 能独立写 + 排查问题 | 熟练使用 |
| 了解 | 写过 demo 或读过文档 | 了解、有使用经验 |

> 不要把 "Redis" 写成精通——多半被一句「讲下 Redis 单线程为什么快」秒杀。

## 5. 10 个高频追问 + 标准答法（速查）

| # | 提问 | 抓手 |
|---|---|---|
| 1 | 自我介绍 1 分钟 | 「身份 + 当前学习状态 + 1 个最满意的项目（含指标）+ 求职意向」 |
| 2 | 这个项目最难的点 | 选一个**有 trade-off** 的（如 JWT 撤销），讲选项 + 选择 + 后果 |
| 3 | 项目最大瓶颈 | 用「DB / 缓存 / 应用层 / 网络」框架定位，说出量化判断依据 |
| 4 | 为什么不用 Spring Data JPA | 答 MyBatis + SQL 可控、复杂 SQL 友好；不要黑 JPA |
| 5 | HashMap 红黑树阈值为什么是 8 | 泊松分布 + 平均查找成本 + 转换代价 trade-off（12 章） |
| 6 | 线上 CPU 100% 怎么排查 | top -Hp / printf %x / jstack 三板斧（54 章） |
| 7 | Spring 事务为什么会失效 | 自调用 / 异常类型 / 非 public / 数据库引擎，6 个场景（35 章） |
| 8 | 缓存一致性怎么保 | Cache-Aside + 先 DB 后删 + 短 TTL 兜底，必要时延迟双删 |
| 9 | 如果有 100w 用户同时登录 | 限流 + BCrypt cost / 单独服务 + Refresh 分片（59 章） |
| 10 | 你最近在学什么 | 给个有方向的真实答案：「JDK 21 虚拟线程」「ZGC 着色指针」 |

## 7. JDK 23-25 值得关注

| 特性 | 状态 | 价值 |
|---|---|---|
| Virtual Threads（21 GA） | 稳定 | IO 密集吞吐质变 |
| Pattern Matching for switch（21 GA） | 稳定 | 替代大量 instanceof |
| Sequenced Collections（21 GA） | 稳定 | `first()` / `last()` 统一 |
| ScopedValue（25 Preview） | 预览 | 替代 ThreadLocal in VT |
| Stream Gatherers（22+） | 预览/正式 | 自定义中间操作 |
| Foreign Function & Memory API（22 GA） | 稳定 | JNI 替代 |
| Vector API | 长期孵化 | SIMD |

## 8. 进阶路线（结业之后）

```
[当前]   会写 Spring Boot REST API → 进 8-15k 一线后端岗
   ↓
[1 年]   分布式（Spring Cloud / Dubbo），消息队列深用，监控告警
   ↓
[2 年]   微服务 + K8s 部署，CI/CD，性能调优实战
   ↓
[3 年+]  架构方向（中间件 / 高并发 / 存储）or 领域深耕（搜索 / 风控 / 推荐）
```

**别陷阱**：
- 不要堆框架（看到 Sentinel / Seata / SkyWalking 名字就学）→ 抓深一个再下一个。
- 不要追新版本 JDK 但不复习老知识 → HashMap / GC / 事务一直是面试重点。
- 不要只写 demo 不写测试 / 不部署 / 不写文档 → 「能跑」≠「能用」。
