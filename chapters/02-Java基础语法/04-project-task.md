# Java 基础语法 项目任务

## 业务背景

你在一家零售公司做后端开发，产品经理提出需要一个运行在内部终端的"营业额日报计算器"CLI 工具。该工具供收银员在每日收工后手动录入当天各商品的销售数据，系统自动计算营业额、利润、税费，并生成一份简洁的日报摘要。

这个工具未来可能会被集成到 Spring Boot 后端的定时任务中，所以代码结构要清晰、逻辑要可测试。

---

## 技术要求

1. 使用 `Scanner` 从控制台循环录入商品信息（名称、数量、单价），输入 `done` 结束录入
2. 使用基本类型（`int`、`double`）存储和计算数据，注意浮点数除法与整数除法的区别
3. 使用 `if-else` 或 `switch` 对营业额区间进行分级评价（见下方说明）
4. 使用 `for` 或 `while` 实现循环逻辑
5. 结果使用 `printf` 格式化输出，金额保留 2 位小数

---

## 任务说明

**录入流程**：
1. 每轮输入一个商品名称（字符串）；若输入 `done` 则结束录入
2. 输入该商品的销售数量（int，必须 > 0，否则提示重新输入）
3. 输入该商品的单价（double，必须 > 0，否则提示重新输入）

**计算规则**：
- 每件商品销售额 = 数量 × 单价
- 总营业额 = 所有商品销售额之和
- 税费 = 总营业额 × 13%（增值税）
- 税后净收入 = 总营业额 - 税费
- 利润率评级：
  - 总营业额 < 1000：一般
  - 1000 ≤ 总营业额 < 5000：良好
  - 5000 ≤ 总营业额 < 20000：优秀
  - 总营业额 ≥ 20000：卓越

**输入/输出示例**：

```
=== 日报录入系统 ===
输入商品名称（输入 done 结束）：可乐
数量：50
单价：3.5
可乐 销售额：175.00 元

输入商品名称（输入 done 结束）：薯片
数量：30
单价：8.0
薯片 销售额：240.00 元

输入商品名称（输入 done 结束）：done

===== 日报摘要 =====
商品数：2
总营业额：415.00 元
税费（13%）：53.95 元
税后净收入：361.05 元
今日评级：一般
====================
```

---

## 验收标准

- [ ] 输入数量或单价为 0 或负数时，程序提示错误并要求重新输入，不崩溃
- [ ] 当没有录入任何商品就输入 `done` 时，程序打印"今日未录入任何销售数据"并退出
- [ ] 税费计算正确（总营业额 × 0.13），金额输出均保留 2 位小数
- [ ] 评级逻辑正确（四个区间对应四个等级）
- [ ] 代码中不使用任何数组或集合（本章未学），只用基本类型和循环累加

---

## 提示（不是答案）

1. 不用数组也能完成本题：用变量 `totalRevenue`（double）循环累加每次的销售额，用 `itemCount`（int）计数即可，不需要存储每条记录的细节
2. 输入校验可以用 `do-while` 或 `while(true)` + `break` 的方式实现"输入合法才继续"的循环

## 常见坑

1. **Scanner.nextLine() 读到空字符串**：在使用 `nextInt()` 或 `nextDouble()` 读取数字后，缓冲区残留了换行符，紧接着调用 `nextLine()` 读取商品名称时会直接读到空字符串。解决方式：每次读完数字后，调用一次 `scanner.nextLine()` 清除换行符

2. **double 精度导致税费计算偏差**：本任务允许使用 double 计算，但如果你用 `415 * 0.13` 发现结果是 `53.949999...`，这是正常的浮点问题，使用 `printf("%.2f", ...)` 格式化输出即可展示正确结果；若追求精确，可改用 `BigDecimal`

3. **营业额为 0 时的边界判断**：如果没有录入任何商品就结束，`totalRevenue` 是 0，此时不应走入评级逻辑，应该提前返回并打印特殊提示
