# 错题与易错点

> 任何「看着懂、写不对 / 被反问答不上来」都记进来。每条至少包含**根因**（不是只记现象），否则复习时无法举一反三。

## 字段约定

- **日期**：YYYY-MM-DD，记录犯错那天。
- **章节**：`NN-章节名` 或 `interview-bank#题号`。
- **现象**：我写了什么 / 说了什么 / 报了什么错。
- **根因**：底层原因。比「忘了加 `@Transactional`」更深一层，例如「同类自调用 → AOP 代理绕过」。
- **修复**：最小代码 / 配置改动。
- **下次复习**：完成日 + 3 / +7 / +30 三个日期循环（同 `[[review-plan]]` 节奏）。
